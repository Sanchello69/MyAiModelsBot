# Настройка API ключа OpenRouter

## ✅ API ключ уже настроен!

Ключ уже добавлен в файл `local.properties` и автоматически используется приложением.

### Как это работает:

1. **API ключ хранится в `local.properties`**
   - Файл находится в корне проекта
   - Автоматически игнорируется Git (защищен от коммита)
   - Ключ: `OPENROUTER_API_KEY`

2. **Gradle читает ключ при сборке**
   - В `app/build.gradle.kts` настроено чтение из `local.properties`
   - Ключ добавляется в `BuildConfig.OPENROUTER_API_KEY`

3. **Приложение использует ключ через Koin**
   - `NetworkModule.kt` читает ключ из `BuildConfig`
   - Автоматически добавляется в HTTP заголовки

## Для других разработчиков

Если вы клонируете проект, добавьте свой ключ в `local.properties`:

### Шаг 1: Получение API ключа

1. Перейдите на [OpenRouter](https://openrouter.ai/)
2. Создайте аккаунт или войдите
3. Перейдите в раздел [Keys](https://openrouter.ai/keys)
4. Создайте новый API ключ

### Шаг 2: Добавление ключа

Откройте или создайте файл `local.properties` в корне проекта и добавьте:

```properties
OPENROUTER_API_KEY=ваш-ключ-здесь
```

### Шаг 3: Синхронизация Gradle

После добавления ключа:
1. Sync Project with Gradle Files
2. Rebuild проекта

## Проверка работы

После настройки ключа:
1. Запустите приложение
2. Отправьте тестовое сообщение
3. Если все настроено правильно, вы получите ответ от AI

При ошибках проверьте:
- Правильность ключа
- Наличие интернет-соединения
- Logcat для просмотра сообщений об ошибках
